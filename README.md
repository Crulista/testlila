# LILA BLACK — Player Journey Visualizer

A web-based visualization tool that lets Level Designers explore player behavior across LILA BLACK's three maps. Built for the LILA Games product assignment.

**🔗 Live Demo:** [Deployed URL will go here]

## Features

- **Player Journey Visualization** — View individual player paths on the correct minimap with world-to-pixel coordinate mapping
- **Human vs Bot Distinction** — Humans shown in color, bots in grey with reduced opacity
- **Event Markers** — Kills (⚔️), Deaths (💀), Storm Deaths (🌩️), Loot (📦) rendered as distinct markers
- **Filtering** — Filter by map, date, and search by match ID
- **Timeline Playback** — Watch a match unfold over time with play/pause, scrub, and 0.5x–10x speed control
- **Heatmap Overlays** — Toggle kill zones, death zones, storm deaths, loot spots, and traffic density across the full dataset
- **Match Browser** — Browse all 796 matches with metadata (player count, kills, deaths, duration)
- **Player List** — See all players in a selected match with color-coded paths

## Local Setup

### Prerequisites

- Node.js 18+
- Python 3.8+ (for data processing)
- The `player_data.zip` dataset

### Step 1: Process the Data

```bash
# Install Python dependencies
pip install pyarrow pandas

# Place player_data/ directory at project root (from the zip)
# Run the processor
python scripts/process_data.py
```

This generates JSON files in `public/data/`:
- `match_index.json` — match metadata for the sidebar
- `heatmaps.json` — aggregated spatial data per map
- `daily_stats.json` — per-day statistics
- `matches/<id>.json` — detailed data per match (lazy-loaded)

### Step 2: Copy Minimap Images

```bash
cp player_data/player_data/minimaps/* public/minimaps/
```

### Step 3: Run the Frontend

```bash
npm install
npm run dev
```

Open `http://localhost:5173` in your browser.

### Step 4: Build for Production

```bash
npm run build
```

Output goes to `dist/` — deploy this folder to any static host.

## Deployment

Deployed on Vercel:

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel --prod
```

Or connect the GitHub repo to Vercel for auto-deployment.

## Coordinate Mapping

World coordinates `(x, z)` are mapped to minimap pixels using:

```
u = (x - origin_x) / scale
v = (z - origin_z) / scale
pixel_x = u * 1024
pixel_y = (1 - v) * 1024    ← Y-flipped for image coordinates
```

The `y` column represents elevation and is not used for 2D mapping. See ARCHITECTURE.md for full details.
